1、git实操

① 创建版本库

​x
git  init()  --初始化
git config user.name "yrx"
git config user.email "yrx@qq.com"
​
--global 表示全局属性，所有的git项目都会共用属性（如果不想用全局的就把global去掉）

② 提交文件

xxxxxxxxxx
1、查看文件状态：                        
git  status  
2、将文件添加到暂存区：                    
git  add  文件名
3、提交文件到本地库（直接带注释提交）            
git  commit  –m “注释内容”
​
​

③ 查看文件提交记录

xxxxxxxxxx
git log  文件名                                --查看历史记录
git log  --pretty=oneline 文件名               --简易信息查看
​

④ 回退历史

xxxxxxxxxx
git  reset  --hard HEAD^                        --回退到上一次提交
git  reset  --hard HEAD~n                      --回退n次操作

⑤ 版本穿越

xxxxxxxxxx
git  reflog  文件名                             --查看历史记录的版本号
git  reset  --hard  版本号                     --执行可以到这个版本

⑥ 还原文件

xxxxxxxxxx
git  checkout -- 文件名                           --还原文件（在本地将文件盖卵后可通过它来还原）

⑦ 删除某个文件

xxxxxxxxxx
先删除文件
再git add 再提交

2、分支

① 创建分支

xxxxxxxxxx
git  branch  <分支名>
git branch –v  查看分支

② 切换分支

xxxxxxxxxx
git checkout  <分支名>
一步完成： git checkout  –b  <分支名>

③ 合并分支（改变谁，切换到谁上）

xxxxxxxxxx
先切换到主干   git  checkout  master
git  merge  <分支名>

4、远程地址

① 增加远程地址

xxxxxxxxxx
git remote add  <远端代号>   <远端地址> 
例：git  remote  add  origin  https://github.com/user111/Helloworld.git

② 推送到远程库

xxxxxxxxxx
git  push   <远端代号>    <本地分支名称>。
例： git  push  origin  master
​
注：之前需要
git add         
git commit

③ 从远程仓库克隆

xxxxxxxxxx
解释：克隆的默认远端代号为origin
git  clone   <远端地址>   <新项目目录名>。
例 git  clone  https://github.com/user111/Helloworld.git   hello_world

④ 从远程仓库更新项目

xxxxxxxxxx
git  pull   <远端代号>   <远端分支名>。
例 git pull origin  master

6、协作冲突

在上传或同步代码时，由于你和他人都改了同一文件的同一位置的代码，版本管理软件无法判断究竟以谁为准，就会报告冲突,需要程序员手工解决。

解决冲突办法：

x
1、修改合并
2、git add 
3、git commit
